# Backend (NestJS + Prisma)

## Run
```
npm i
# DATABASE_URL in .env must point to Postgres
npx prisma migrate dev -n init
npm run start:dev
```

## Env
- `DATABASE_URL=postgresql://postgres:postgres@localhost:5432/referral?schema=public`
- `PORT=3000` (optional)

## Auth
- `x-user-id` header required for all endpoints (fake guard for demo)

## Endpoints
- POST `/api/referral/generate` → `{ code }`
- POST `/api/referral/register` `{ code }` → `{ level }`
- GET  `/api/referral/network` → `{ level1, level2, level3 }`
- GET  `/api/referral/earnings` → `{ total, byLevel }`
- DEV: POST `/api/trades/mock` `{ tradeId, userId, feeAmount, token? }` → `{ ok: true }`

## DDD / Hex / CQRS
- `src/domain`: value-objects, policies, domain services
- `src/application`: ports (interfaces), tokens, app services (orchestrators)
- `src/infrastructure/prisma`: PrismaService, repository adapters
- `src/interfaces/http`: controllers + DTOs

## Commission Policy
- Default: cashback to trader (if any), uplines: 30%, 3%, 2% for Level 1..3
- Idempotent by unique `(beneficiaryId, sourceTradeId, level)`

## Prisma Models
- `User`, `ReferralLink`, `Trade`, `CommissionLedgerEntry`, `IdempotencyKey`

## Testing
```
npm test
```
Covered: Money, Percentage, DefaultPolicy, CommissionService, ReferralService, ReferralAppService, TradesAppService.

## Error Handling & Limits
- Global ValidationPipe
- Global HttpExceptionFilter → `{ code, message, path }`
- LoggingInterceptor (request latency)
- `@nestjs/throttler` (basic rate limiting)

## Future Work
- Outbox and MerkleDistributorPort adapter (on-chain distribution)
- e2e tests for HTTP layer
- Admin endpoints (recompute, reports)
